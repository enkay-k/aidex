# Module Doc Format

Use this order for every file doc in `docs/modules/`:

1. `# Module: <repo-relative-path>`
2. `Source file` link to the code file
3. `Human HTML view` link to generated docs/site page
4. `## Purpose`
5. `## Flow Diagram` with Mermaid
6. `## Functions and Responsibilities` table
7. `## Dependencies`
8. `## Related Documentation`

## Writing rules

- Explain behavior from product/system perspective, not only code internals.
- Keep function descriptions one line each.
- Link to upstream and downstream modules.
- Keep diagrams current when behavior changes.
